# Automated Test Plan — Design Spec

**Date:** 2026-07-12
**Status:** Approved (design), pending implementation plan
**Scope:** Comprehensive automated test strategy + full-chain logging for Mini Capsule (macOS)

## 1. Context & Goals

Mini Capsule is a macOS clipboard manager (MVVM + Services, SwiftUI + AppKit host,
SwiftData persistence, Carbon global hotkeys, CGEvent paste). It currently has
**104 unit tests** (Swift Testing) across 12 files plus 2 XCUITest files. Coverage
is solid for pure logic but **thin/indirect** for pasteboard reading and **absent**
for permission-gated paths (CGEvent paste, Carbon hotkeys, NSPanel behavior).

The user's five requirements:

1. Cover **all** functionality.
2. Cover **all** boundary/edge cases.
3. **No** functionality omitted.
4. When a problem appears, it must be **traceable back to root cause** (回溯).
5. Save **full-chain logs** for locating causes.

### Resolved decisions (from brainstorming)

| Fork | Decision |
|---|---|
| Where logging lives | **App + tests** — a structured logging layer in the app itself, plus tests that assert on and archive those logs. |
| Code that can't run headlessly | **Refactor + manual gate** — extract logic behind protocol seams so ~all logic runs headless; the irreducible residue gets a documented manual checklist. |
| Meaning of 回溯 | **Log-based root-cause** — archived full-chain logs + per-failure timeline report locate the cause without re-running. (Regression-test-on-bug is ordinary hygiene, not a separate pillar.) |
| Architecture | **A: Seams + logging facade + tiered test plan** — protocol seams, `os.Logger`-backed facade with injectable test sink + file archival, Unit/Integration/UI tiers via `.xctestplan`, `run-tests.sh` + optional GitHub Actions. |

## 2. Coverage Philosophy — what "no omissions" means

Because parts of the app cannot run headlessly, the no-omissions guarantee is made
**auditable** rather than aspirational:

> **Every behavior is covered by exactly one of: an automated test, or a numbered
> manual-checklist item.** If something can't be automated, it becomes a checklist
> entry with a documented reason. A **traceability matrix** (feature → test IDs /
> checklist IDs) is the proof artifact, and a **meta-check test** fails the build if
> any inventory row has zero linked coverage.

## 3. The 4 Test Tiers

| Tier | Runs | Host | Covers |
|---|---|---|---|
| **T1 Unit** | headless, fast, deterministic | no app, no window server | Pure logic: parsing, dedup, cap math, image transforms, settings codec, log facade |
| **T2 Integration** | headless | in-memory SwiftData + fake seams | Full flows: capture→dedup→store→notify, paste write path, hotkey register/dispatch, settings side-effects |
| **T3 UI** | windowed | XCUITest, real app | User-visible flows: capsule hover/expand, list nav, search, context menu, settings windows |
| **T4 Manual** | human | real machine w/ permissions | Irreducible residue: real CGEvent Cmd+V into another app, real global hotkey, real NSPanel compositing/multi-space |

## 5. Logging Subsystem

New files under `Mini Capsule/Logging/`.

```swift
enum LogCategory: String { case capture, dedup, store, paste, hotkey,
                                settings, window, menubar, cleanup, ui, app }
enum LogLevel: Int, Comparable { case debug, info, notice, warning, error, fault }

struct LogEvent {
    let timestamp: Date
    let category: LogCategory
    let level: LogLevel
    let message: String
    let metadata: [String: String]   // counts/types/ids ONLY — never content
    let correlationID: String?       // ties one capture/paste chain together
}

protocol LogSink: Sendable { func write(_ event: LogEvent) }

final class Log {                    // façade: fans out to N sinks
    static let shared = Log(sinks: [OSLogSink(), FileSink()])
}
```

### Sinks

| Sink | Role |
|---|---|
| `OSLogSink` | wraps `os.Logger(subsystem:category:)` → Console.app / `log stream` / production debugging |
| `FileSink` | appends **JSONL** to `~/Library/Application Support/Mini Capsule/logs/` (rotating, size-capped) → archivable full chain from real sessions |
| `InMemoryLogSink` | test-only: records `[LogEvent]` in order; tests assert on the sequence; feeds the failure report |

### Correlation ID = "全链路" (full-chain)

Each capture cycle and each paste operation gets a short `correlationID`. Every event
in that operation carries it, so filtering the archive by one ID reconstructs the
entire chain:

```
[cap a1b2] poll tick interval=0.5
[cap a1b2] changeCount 41→42, suppressed=false
[cap a1b2] readPasteboard tier=image type=image bytes=48213
[dedup a1b2] md5=9f3c… miss
[store a1b2] enforceCap removed=1 ids=[…]  insert id=E7…  save=ok
[cap a1b2] posted .clipItemsDidChange
```

### Two hard rules

1. **Privacy — never log clipboard content.** Only lengths, types, md5 prefixes,
   counts, ids. `OSLogSink` also applies `.private` annotations as a second guard.
   This is non-negotiable for a clipboard manager.
2. **No silent failures.** Every `try?` that currently swallows an error (~10 across
   `ClipboardMonitor`/`PasteService`/`SettingsStore`) becomes a logged `.error`
   event. This robustness upgrade ships with the logging work.

## 6. 回溯 — the per-failure root-cause report

Decoupled from fragile "did this test fail" detection inside the test body:

- **Always capture:** every T1/T2 test runs its logic through an `InMemoryLogSink`; a
  helper writes the ordered chain to `TestResults/<run>/logs/<testID>.jsonl`. Fakes
  (pasteboard, key injector, clock) also record their calls into the same artifact.
- **Runner promotes failures:** after the run, `run-tests.sh` parses the `.xcresult`
  for failing tests and copies each failing test's `.jsonl` + assertion message +
  seam-call recording into `TestResults/<run>/failures/<testID>.log`.
- **UI tier (T3):** the app's `FileSink` path is set via launch environment; logs
  attach to the `.xcresult` as `XCTAttachment`, so a UI failure carries its app-side
  chain too.

Net: open one `failures/<testID>.log` and see the broken assertion **plus** the full
event timeline and the exact fake inputs that produced it — no re-run needed.

## 7. Refactor Seams

Guiding principle (low-risk): **every seam is injected with a production default
pointing at the real singleton**, so call sites and runtime behavior are unchanged;
tests pass a fake.

```swift
init(settings: SettingsProtocol,
     pasteboard: PasteboardReading     = NSPasteboard.general,
     workspace:  FrontmostAppProviding = NSWorkspace.shared,
     scheduler:  Scheduling            = RealScheduler(),
     selfPaste:  SelfPasteTracker      = .shared,
     log:        LogSink               = Log.shared) { … }
```

### Seams

| Protocol | Wraps (production) | Fake enables | Used by |
|---|---|---|---|
| `PasteboardReading` | `NSPasteboard.general` (changeCount, types, data/string/propertyList/readObjects) | feed any pasteboard state | ClipboardMonitor |
| `PasteboardWriting` | `NSPasteboard.general` (clear/setString/setData/writeObjects) | assert exactly what was written | PasteService |
| `FrontmostAppProviding` | `NSWorkspace.shared.frontmostApplication` | control source-app bundleID/name | ClipboardMonitor |
| `Scheduling` | `Timer` + `DispatchQueue.main.asyncAfter` | advance time deterministically (no real waits) | Monitor burst/poll, CapsuleViewModel delays |
| `AccessibilityChecking` | `AXIsProcessTrustedWithOptions` | test granted & denied branches | PasteService |
| `KeyInjecting` | the CGEvent Cmd+V dance | record "paste fired" without the real HID tap | PasteService |
| `HotKeyRegistering` | Carbon `RegisterEventHotKey` | assert registrations + simulate a fire | HotKeyCenter |

### Two extractions that unblock the biggest gaps

1. **`SelfPasteTracker`** — pull the suppression set out of the all-static
   `PasteService`. Decouples `ClipboardMonitor` from `PasteService` static state, lets
   `PasteService` become an instance with injected seams, and makes suppression
   testable in isolation (including the 200-entry overflow reset).
2. **Decompose `checkPasteboard()`** into a pure decision + effects:

   ```swift
   func classify(_ pb: PasteboardReading) -> CaptureOutcome        // returns a value
   enum CaptureOutcome { case suppressed, skipped(reason:), dedupHit(id:), insert(ClipItem) }
   func apply(_ outcome: CaptureOutcome, context:)                  // insert/save/notify + logs
   ```

   T2 asserts on the `CaptureOutcome`, the resulting store state, **and** the emitted
   log chain — three independent checks per flow.

### Files touched

| Action | Files |
|---|---|
| **New** | `Logging/*` (facade + sinks), `Services/Seams/*` (protocols + prod conformances + `SelfPasteTracker`) |
| **Modified (behavior-preserving)** | `ClipboardMonitor.swift`, `PasteService.swift`, `HotKeyCenter.swift`, `CapsuleViewModel.swift`, `SettingsStore.swift` — inject seams, add logging, make `readPasteboard` internal |
| **Unchanged** | Models, ColorHex, most UI views (exercised at T3/T4) |

### Risk control

Every refactor step is **test-first (TDD)**: write the T2 test against the new seam,
watch it fail, refactor until green, confirm the pre-existing 104 tests still pass.
Production-default parameters mean a half-finished refactor still behaves identically.

## 8. Test Organization

`MiniCapsule.xctestplan`:

- **Shared settings:** code coverage **ON, scoped to the app target**; env
  `MC_TEST_LOG_DIR` → the run's results dir.
- **T1/T2** in `Mini CapsuleTests`, separated by folder + Swift Testing suites
  (`Unit/`, `Integration/`) and `@Tag`s (`.unit`, `.integration`).
- **T3** = `Mini CapsuleUITests`.
- **Configs:** `fast` (T1+T2, every change) and `full` (+T3). T4 is the manual doc.

## 9. Traceability Matrix

`docs/testing/traceability-matrix.md`: a machine-parseable table with one row per
inventory item (§4) — it is the single source of truth and enumerates every inventory
row — → linked **test IDs** (`Suite/function`) and/or **checklist IDs** (`M#`).

A **meta-check test** parses the matrix table and **fails if any row has zero links**,
mechanically enforcing requirement #3. (A row's presence is asserted against the §4
census so the matrix itself can't silently drop a feature.)

## 10. Runner & Results

`Scripts/run-tests.sh`:

```
xcodebuild test -testPlan MiniCapsule \
  -resultBundlePath TestResults/<ts>/result.xcresult -enableCodeCoverage YES
  → MC_TEST_LOG_DIR set so each test's full chain lands in logs/<testID>.jsonl
  → parse .xcresult (xcresulttool/xccov): failures + coverage%
  → promote each failing test's .jsonl → failures/<testID>.log (+ assertion + fake-call recording)
  → write summary.md; exit ≠0 on any failure, coverage < gate, or unlinked matrix row
```

Results layout:

```
TestResults/2026-07-12-1530/
├── result.xcresult
├── logs/<testID>.jsonl        # full chain, ALWAYS
├── failures/<testID>.log      # promoted on failure — open this to 回溯
├── coverage.json
└── summary.md
```

`TestResults/` → `.gitignore` (alongside `DerivedData/`).

### Coverage gate (realistic, scoped)

Applies to **logic files** (`Services/`, `UI/*ViewModel`, `Settings/`, `Utilities/`,
`Logging/`) at **85%**. Pure SwiftUI view files are **excluded** from the number —
covered at T3/T4, not by line count — so the gate isn't gamed on view bodies.

## 11. CI (optional)

`.github/workflows/tests.yml` on a macOS runner → `run-tests.sh fast` (T1+T2 only; UI
excluded as flaky) → upload `TestResults/` as a build artifact. Delivers "run on every
change + archived logs" without a local machine.

## 12. Manual Checklist (T4)

`docs/testing/manual-checklist.md` — numbered items, each with steps + expected result
+ the inventory row it closes.

| ID | Verifies |
|---|---|
| M1–M2 | real Cmd+V lands text / image in another app |
| M3–M4 | global hotkey show-hide / quick-paste while another app is focused |
| M5–M6 | capsule floats above all windows + joins Spaces; drag frame persists across relaunch |
| M7 | Accessibility denied → paste no-ops gracefully (and logs `.error`) |
| M8–M9 | launch-at-login; menu-bar recent-items paste |

## 13. Cadence

- **Every change:** T1+T2 (`run-tests.sh fast`, local and/or CI) — fully automated,
  deterministic.
- **Pre-release:** T3 + full T4 checklist.
- **New bug found:** reproduce with a failing T1/T2 test first, then fix — the log
  chain in `failures/` tells you where to put the test.

## 14. Non-Goals / Out of Scope

- Rewriting UI views for testability beyond what T3/T4 need.
- Moving logic into an SPM package (Approach C) — deferred; SwiftData `@Model` types
  resist the module boundary today.
- A permissioned CI runner that actually fires CGEvent/global hotkeys — the manual
  checklist owns that residue.
- Performance/load/memory-budget testing (tracked separately in the memory-budget
  plan).

## 15. Risks & Mitigations

| Risk | Mitigation |
|---|---|
| Seam refactor changes runtime behavior | Production-default params + TDD + keep 104 existing tests green each step |
| `os.Logger` retrieval in tests is awkward | Tests use `InMemoryLogSink`, not OSLogStore; `OSLogSink` is production-only |
| Accidental clipboard-content logging (privacy) | Metadata-only rule + `.private` annotations + a lint/test asserting no content fields in `LogEvent.metadata` |
| Coverage gate gamed on view bodies | Gate scoped to logic files; views excluded |
| UI tests flaky on CI | T3 excluded from the required `fast` gate; run in `full`/pre-release |
